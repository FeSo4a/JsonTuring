# JsonTuring

一个基于自定义 JST（JsonScript Turing）语言的图灵机模拟器。通过简洁的配置文件定义图灵机的状态转换规则，可视化地观察图灵机的运行过程。

## 项目简介

JsonTuring 是一个教育性和实验性的图灵机模拟工具，它使用自定义的 JST 脚本语言来描述图灵机的行为。程序会解析 JST
文件，初始化纸带，然后按照定义的规则逐步执行，实时显示纸带状态和读写头位置。

### 主要特性

- 自定义 JST 脚本语言，语法简洁直观
- 支持完整的图灵机操作：状态转换、数值修改、读写头移动
- 实时可视化显示纸带状态和读写头位置
- 支持清屏、暂停、延时等调试功能
- 模块化架构设计，包含代码解析、执行引擎和屏幕控制三个核心模块
- 支持将 JST 文件编译为独立的可执行文件

## JST 语言语法

JST（JsonScript Turing）是一种专为图灵机设计的领域特定语言（DSL），包含三个核心块：init、fn 和 main。

### 基本语法结构

```jst
// 这是注释，使用双斜杠
// 初始化块：定义图灵机的初始状态 
init {
    head A; // 初始表头（状态标识符） 
    value 1; // 初始格子数值 
    pos 5; // 读写头初始位置 
    max 10 // 纸带长度（最后一项不需要分号） 
}
// 函数定义块：定义状态转换规则 
fn 操作名称 { 
    if ( 
        head = A; // 触发条件：当前表头为 A 
        value = 1 // 触发条件：当前格子值为 1 
    ) 
    head B; // 执行动作：将表头改为 B 
    value 0; // 执行动作：将当前格子值改为 0 
    move L // 执行动作：读写头向左移动（L左/R右/N不动/H停机） 
}
// 主程序块：定义操作的执行顺序 
main {
    操作名称1; 
    操作名称2;
    ...
}
```

### 语法详解

#### 1. init 块（初始化）

定义图灵机的初始配置，包含四个参数：

- `head`：表头标识符，用于区分不同的状态（如 A、B、C）
- `value`：纸带格子的初始值（通常为 0 或 1）
- `pos`：读写头的起始位置（从 0 开始计数）
- `max`：纸带的总长度

#### 2. fn 块（函数定义）

定义状态转换规则，每个函数包含：

- 触发条件（if 块）：
    - `head`：判断当前表头是否匹配
    - `value`：判断当前格子值是否匹配

- 执行动作：
    - `head`：设置新的表头值
    - `value`：修改当前格子的值
    - `move`：指定读写头移动方向
        - L：向左移动（Left）
        - R：向右移动（Right）
        - N：不移动（None）
        - H：停机（Halt）

#### 3. main 块（主程序）

定义操作的执行顺序。程序会按顺序尝试每个操作，当某个操作的触发条件满足时立即执行，然后进入下一轮循环。

### 执行流程

1. 读取 init 块，初始化纸带和状态
2. 进入 main 块，按顺序检查每个操作
3. 如果某个操作的触发条件（head 和 value）都满足，则执行该操作
4. 执行完毕后，立即开始下一轮循环（跳过后续操作）
5. 重复步骤 2-4，直到遇到 move H 停机指令

## 使用方法

### 运行 JST 文件

运行示例文件：

```bash
cd src
python JsonTuring.py -r=../example/example
```

注意：文件路径不需要 .jst 后缀。

### 查看代码解析结果

```bash
python src/JsonTuring.py -a=../example/example
```

这会显示分词后的代码结构和提取到的函数定义，适合调试和学习。

### 命令行参数

```bash
python src/JsonTuring.py --help
```

选项：

* -r, --run FILE 运行指定的 .jst 文件（无需后缀）
* -a, --ast FILE 显示分词后的 .jst 文件内容（无需后缀）
* -c, --clear 每个步骤结束后清空屏幕（默认不清屏）
* -t, --time SECONDS 每个步骤结束后停顿的秒数（默认为 0）
* -p, --pause 每个步骤结束后等待用户按回车键继续

## 示例程序说明

项目提供了示例文件 `example/example.jst`，演示了一个简单的状态转换：

1. 初始状态：表头 A，当前位置值为 1
2. 第一个操作（one）：如果表头是 A 且值为 1，则将表头改为 B，值改为 0，向左移动
3. 第二个操作（two）：如果表头是 B 且值为 0，则将表头改回 A，值改为 1，然后停机

运行该示例可以观察到图灵机从启动到停机的完整过程。

## 编写自己的 JST 程序

### 设计建议

1. 为不同的状态设计清晰的表头命名
2. 确保每个状态都有对应的转换规则
3. 至少要有一个停机条件（move H）
4. 注意处理边界情况（读写头移动到纸带两端）
5. 使用注释说明每个操作的作用

## 开发说明

### 模块介绍

- **code_ast**：负责 JST 代码的词法分析和语法解析
    - `ast.py`：核心解析器，将文本转换为结构化数据
    - `load_jst.py`：文件加载和初始化参数提取

- **run_jst**：图灵机的执行引擎
    - `run_code.py`：实现图灵机的运行循环和状态转换

- **screen**：终端控制
    - `clear_screen.py`：跨平台清屏功能

### 扩展开发

如需添加新功能，可以参考以下方向：

1. 在 `ast.py` 中扩展语法解析规则
2. 在 `run_code.py` 中添加新的操作类型
3. 在 `screen` 模块中增强显示效果

## 常见问题

### 1. 运行时出现 IndexError: list index out of range

这通常是因为 fn 块中的触发条件或执行动作格式不正确。确保：

- if 块中的条件和执行体都是成对的键值对
- 每个配置项都使用正确的语法（key = value）

### 2. 程序无法正常停机

检查是否定义了 move H 的操作，并确保该操作的触发条件能够被满足。

### 3. 读写头移动超出纸带范围

程序会自动处理边界情况：向左移出左边界会跳转到右端，反之亦然。
