> 项目简介

JsonTuring 是一个教育性和实验性的图灵机模拟工具，它使用自定义的 JST（JsonScript Turing）脚本语言来描述图灵机的行为。程序会解析 JST 文件，初始化纸带，然后按照定义的规则逐步执行，实时显示纸带状态和读写头位置。

> 功能特性

自定义JST语言

语法简洁直观的领域特定语言

完整图灵机操作

支持状态转换、数值修改、读写头移动

可视化显示

实时显示纸带状态和读写头位置

调试功能

支持清屏、暂停、延时等调试选项

模块化架构

代码解析、执行引擎、屏幕控制分离

编译为EXE

可将JST文件打包为独立可执行文件

> JST 语言语法

基本语法结构

// 这是注释，使用双斜杠
// 初始化块：定义图灵机的初始状态
init {
    head A;      // 初始表头（状态标识符）
    value 1;     // 初始格子数值
    pos 5;       // 读写头初始位置
    max 10       // 纸带长度（最后一项不需要分号）
}

// 函数定义块：定义状态转换规则
fn 操作名称 {
    if (
        head = A;    // 触发条件：当前表头为A
        value = 1    // 触发条件：当前格子值为1
    )
    head B;          // 执行动作：将表头改为B
    value 0;         // 执行动作：将当前格子值改为0
    move L           // 执行动作：读写头向左移动（L左/R右/N不动/H停机）
}

// 主程序块：定义操作的执行顺序
main {
    操作名称1;
    操作名称2
}

语法详解

块类型 参数 说明
init
（初始化）		 head 表头标识符，用于区分不同的状态（如A、B、C）
value 纸带格子的初始值（通常为0或1）
pos 读写头的起始位置（从0开始计数）
max 纸带的总长度
fn
（函数定义）		 触发条件（if块）：
head 判断当前表头是否匹配
value 判断当前格子值是否匹配
执行动作：
head 设置新的表头值
value 修改当前格子的值
move L向左 / R向右 / N不动 / H停机
main
（主程序）		 定义操作的执行顺序，按顺序尝试每个操作

> 使用方法

1. 运行JST文件

# 运行示例文件
cd src
python JsonTuring.py -r=../example/example

# 注意：文件路径不需要.jst后缀

2. 查看代码解析结果

# 显示分词后的代码结构和函数定义
python src/JsonTuring.py -a=../example/example

3. 命令行参数

参数 说明 默认值
-r, --run FILE 运行指定的.jst文件（无需后缀） -
-a, --ast FILE 显示分词后的.jst文件内容（无需后缀） -
-c, --clear 每个步骤结束后清空屏幕 不清屏
-t, --time SECONDS 每个步骤结束后停顿的秒数 0
-p, --pause 每个步骤结束后等待用户按回车键继续 不暂停

4. 使用示例

# 基本运行
python src/JsonTuring.py -r=../example/example

# 带清屏和延时运行（每步延迟1秒）
python src/JsonTuring.py -r=../example/example -c -t 1

# 单步调试模式（每步等待用户确认）
python src/JsonTuring.py -r=../example/example -p

# 查看解析结果
python src/JsonTuring.py -a=../example/example

> 示例程序说明

// example/example.jst 演示简单的状态转换
init {
    head A;
    value 1;
    pos 5;
    max 10
}

fn one {
    if (
        head = A;
        value = 1
    )
    head B;
    value 0;
    move L
}

fn two {
    if (
        head = B;
        value = 0
    )
    head A;
    value 1;
    move H
}

main {
    one;
    two
}

执行流程：

  • 初始状态：表头A，当前位置值为1
  • 第一个操作（one）：如果表头是A且值为1，则将表头改为B，值改为0，向左移动
  • 第二个操作（two）：如果表头是B且值为0，则将表头改回A，值改为1，然后停机

> 编写自己的JST程序

设计建议：
  • 为不同的状态设计清晰的表头命名
  • 确保每个状态都有对应的转换规则
  • 至少要有一个停机条件（move H）
  • 注意处理边界情况（读写头移动到纸带两端）
  • 使用注释说明每个操作的作用

> 常见问题

1. 运行时出现 IndexError: list index out of range

这通常是因为fn块中的触发条件或执行动作格式不正确。
确保：
  - if块中的条件和执行体都是成对的键值对
  - 每个配置项都使用正确的语法（key = value）

2. 程序无法正常停机

检查是否定义了 move H 的操作，并确保该操作的触发条件能够被满足。

3. 读写头移动超出纸带范围

程序会自动处理边界情况：
  - 向左移出左边界会跳转到右端
  - 向右移出右边界会跳转到左端
